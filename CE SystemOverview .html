01

סקירה כללית

תיאור המערכת

מערכת ועדות חריגים מנהלת את תהליך בקשות ההקלה בהשתתפות עצמית של לקוחות רווחה — ממשלוח הפניה ממחלקות לשירותים חברתיים (מש"ח) ברשויות המקומיות, דרך דיון בוועדת חריגים במחוז, ועד מתן ההחלטה ועדכון הלקוח.

רקע לפיתוח המערכת

המערכת הקיימת אינה נותנת מענה הולם לצרכים הנוכחיים:

  • תקשורת עם מש"ח מתבצעת בדואר רגיל ובטפסים פיזיים — ללא מיחשוב ישיר
  • המערכת הקיימת תומכת רק בסוג פניות אחד (מסגרות לקטינים)
  • מגיעה כמות גדולה של מידע לא רלוונטי — דוחות סוציאליים מלאים
  • קושי בניהול מכסות, מעקב תקציבי ויצירת דוחות מתקדמים
  • אובדן מסמכים ותקשורת מילולית מרובה בין המשרד למש"ח
היקף השימוש
  • 4 מחוזות: צפון, דרום, מרכז, ירושלים — בנוסף למטה
  • סוגי משתמשים: מפקחים מחוזיים, עובדים סוציאליים, עובדי זכאות, מנהלי מש"ח, מדריכי צוות
  • מחלקות לשירותים חברתיים בכלל רשויות המקומיות בישראל
  • גורמי מטה: אחראי ערעורים, אחראי רטרו
קריטיות המערכת

קריטיות גבוהה — המערכת מנהלת החלטות המשפיעות ישירות על תשלומי לקוחות רווחה, כולל קטינים, אזרחים ותיקים ומוגבלויות. כל עיכוב או תקלה עלולים לפגוע במתן שירות לאוכלוסיות רגישות.

  • גישה לשירות WCF בסביבת TEST לבדיקות API
  • נתוני בדיקה (test data) — פרטי לקוחות, פניות, מסגרות
  • תיעוד ה-API של שירות WCF — WSDL / specs
  • תיאור User Flows מלא לכל 4 סוגי הפניות
